What is Less?

Less is a pre-processor for CSS.

Why need Less?

Variables: In CSS, a lot of information is repeated. For example, the same color on various places. When developers want to change them, they have to change all of them. It would be great if we can use variables in CSS.

Mixins: Same set of CSS properties are repeated in different places. Woud be great if we can group them together.

Nested rules: makes it look similar to HTML structure.

Operators and functions: color: #5B83AD + #111 = #6c94be

Namespaces for grouping variables and mixins:

How does Less solve these problems?

Variables

@some-color: #428bca; // only need to change this 

.some-class {
  color: @some-color;
}

.some-background {
  background: @some-color;
}

.some-sidebar {
  background: @some-color;
}

.some-class {
  color: #428bca;
}
.some-background {
  background: #428bca;
}
.some-sidebar {
  background: #428bca;
}

Mixins

Mixins: multiple properties can be grouped together.

Mixins as functions: accept arguments

.demo-btn() {
  text-align: center; // Center button text
  cursor: pointer; // Show pointer when hover on button
  padding: 10px; // 10px padding left, top, right and button
  border: 2px solid;
  border-radius: 25px;
}
.demo-btn-cardinal {
  .demo-btn;
  color: @usc-cardinal;
}
.demo-btn-gold {
  .demo-btn;
  color: @usc-gold;
}


.demo-btn-cardinal {
  text-align: center;
  cursor: pointer;
  padding: 10px;
  border: 2px solid;
  border-radius: 25px;
  color: #990000;
}
.demo-btn-gold {
  text-align: center;
  cursor: pointer;
  padding: 10px;
  border: 2px solid;
  border-radius: 25px;
  color: #ffcc00;
}

Mixin Guards

Conditional mixins (functions)

.demo-btn-with-guards (@color) when (lightness(@color) >= 50%) {
  color: black;
}
.demo-btn-with-guards (@color) when (lightness(@color) < 50%) {
  color: white;
}

.demo-btn-cardinal-with-guard {
  ...
  color: white;
}
.demo-btn-gold-with-guard {
  ...
  color: black;
}

How to use Less

Command line: install Less compiler (npm install less -g), compile .less files to .css files.

Frontend: use less.js, but not recommended.

Backend: supported by Node.js, Java, .NET, PHP, Python and Ruby. http://lesscss.org/about/#ports
